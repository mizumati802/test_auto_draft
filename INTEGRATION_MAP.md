# test_auto_draft 統合・参照マップ

## 🛠 統合元・参照リソース
本プロジェクトのオートメーション構築において核となるリソース群です。

### 1. フロントエンド（拡張機能）
- **google-lens-r2-extension**: 
  - 役割：メルカリ編集画面における「サムネイル画像」の特定ロジック、およびバックエンドエンドポイントへの画像送信・接続方法の参考。
- **easyregister_master_refactor**:
  - 役割：本プロジェクトの**根幹統合元**。UI構造、テンプレート結合、既存の流し込みロジックを継承。

### 2. バックエンド（Flask / Cloud）
- **vintage_extend.py / vintage_scanner.py**:
  - 役割：**画像予測・解析およびガードレール（整合性チェック）の規範**。AIの推論結果をマスターデータと照合し、不該当なものを不採用・デフォルト化する「フィルタリング・ロジック」を継承し、データの品質を担保する。
- **ai_logic.py**:
  - 役割：**新規テスト解析エンドポイントの設置場所**。タイトル生成（Title Refine）、説明文生成（Description Refine）、Vision AIによる属性抽出のコアロジックをここに集約。
- **external_api.py**:
  - 役割：**拡張機能用ゲートウェイ**。Chrome拡張からのリクエストを `ai_logic.py` へ繋ぐ通信経路。

## 📡 通信・実行フロー（構想）
1. **[Extension]**: `/sell/draft/` ページで未記入（3行以内）を検知。
2. **[Extension]**: サムネイル画像を取得し、`external_api.py` のゲートウェイへ送信。
3. **[Backend]**: `external_api.py` 経由で `ai_logic.py` の画像解析（Vision AI）を実行。
4. **[Backend]**: 解析結果に基づき、最適なテンプレート予測・タイトル生成・説明文生成を実施。
5. **[Extension]**: 生成されたデータを商品説明欄（textarea）およびタイトル欄へ自動流し込み。
