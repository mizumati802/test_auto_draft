# 保存ロジック比較解析レポート（最終版）

本ドキュメントは、`save_ok_.js`（成功版）、`content.js`（失敗版）、および `master_mer_assistant`（実績版）における自動保存ロジックの分析に基づき、`content.js` に施された改善を記録したものである。

## 1. 構造的・戦略的比較（改善後）

| 項目 | `save_ok_.js` (成功版) | `content.js` (旧・失敗版) | `content.js` (新・堅牢版) | `master_mer_assistant` (実績版) |
| :--- | :--- | :--- | :--- | :--- |
| **実行タイミング** | 解析開始から一定時間後 | **注入成功フラグ依存** | **注入開始から一定時間後** | 処理完了から約5秒後 |
| **待機時間** | 1.0秒 | 0.5秒 | **3.0秒 + 2.0〜3.0秒 (計5s+)** | 3.0秒 + 2.0〜3.0秒 (計5s+) |
| **保存ボタン特定** | 部分一致 | 完全一致・属性依存 | **実績セレクタ + 部分一致** | 構造的セレクタ (`data-testid`) |
| **安全装置** | なし | 注入判定のみ | **実行直前のデータ存在チェック** | 実行直前の値不一致チェック |

## 2. 導入された改善の核心

### A. 注入判定への依存排除
旧版では「一瞬のDOM遅延で注入失敗と判定されると保存がスキップされる」という致命的な欠陥があった。新版では、注入アクションを実行した後は判定結果に関わらず「保存フェーズ」へ移行し、最終的なデータ存在チェックを経て保存を実行するフローに変更した。

### B. 多重待機による安定化
メルカリのSPA構成では、データ注入後に「保存ボタンが活性化される」まで数秒のラグが発生する。実績版（`master_mer_assistant`）と同じ5秒以上の待機時間を設けることで、このラグを確実に吸収し、未活性状態でのクリックミスを排除した。

### C. 構造的セレクタの採用
`[data-testid="edit-button"] button[type="submit"]` を第一優先にすることで、ページ内に混在する他の「保存」系ボタン（一時保存など）との競合を回避し、確実に「上書き保存」を選択する。

### D. 保護クローズの実装
保存クリックの2秒後にまだ編集画面にいる場合にのみタブを閉じる処理を追加。これにより、正常に保存された場合のみ次のタスク（AutoPilot）へスムーズに移行できる。
