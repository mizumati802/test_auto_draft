## 🚀 プロジェクト目標
画像解析とタイトル解析を並列実行し、5つの重要データを一気通貫で自動生成・流し込みを行う「完全オートメーション出品」の構築。

## 📅 開発フェーズ

### Phase 1: 状態検知とトリガーの確立 (完了)
- [x] 下書き編集ページ（/sell/draft/）の自動検知（SPA対応）。
- [x] アカウント検知（Phase 0）による「通常/古着」モードの自動判定（実装済）。
- [x] ロード3秒後の「未記入判定（3行以内）」実行。
- [x] サムネイル（商品画像）の存在確認。

### Phase 2: パラレル・オートメーション・ロジック
トリガー検知後、以下の2つのRootを実行する。

> ⚠️ **重要：タイトルの役割遷移と順序**
> タイトル欄は最初「情報の入力源（解析用メモ）」として扱われ、Root 🅰️ で分解されます。その後、Root 🅱️ のAI生成によって「Masterタイトル（完成品）」へと上書きされます。したがって、**Root 🅰️ の `remarks` 抽出が完了した後に Root 🅱️ の生成を開始する** という厳密な順序が必要です。

#### 🅰️ タイトルRoot（解析・価格算出）
- [x] 1. **Triple-T パース (t1, t2, t3)**: タイトルを空白で分解し、以下のルールで正規化。
    - **t1 (元値)**: 最初の要素。数値として抽出。
    - **t2 (割引率)**: 2番目の要素。**「数値のみ」かつ「3文字以内」**の場合のみ割引率として採用。
    - **t3 (備考/年代)**: 上記以外（t2条件に外れた2番目以降、または3番目以降の全要素）。
    - *例: 「4000 00s BURBERRY」 → t1:4000, t2:なし, t3:00s BURBERRY*
- [x] 2. **価格算出**: 割引率(t2)を適用した最終販売価格（Price/仕入れ価格）の計算。
- [x] 3. **Remarks抽出**: `t3` を `remarks` (メモ) として抽出。画像解析Rootへの受け渡し準備完了。

#### 🅱️ 画像解析Root（AI生成・マージ）
> ⏳ **待機**: Root 🅰️ 完了から **3秒後** に自動開始（実装済）。
- [x] 1. **ID発行リクエスト**: `purchase_id` をバックエンドから先行取得（実装済）。
- [ ] 2. **Vision解析**: サムネイル画像 + タイトル由来の `remarks` を送信し、属性（C2, C3, 色）を抽出。
    - **ロジック**: `vintage_resolver` のプロンプト生成器を **ラップ(Wrap)** して使用。
    - **識別フラグ**: 解析結果のJSON（context）の `analysis_type` を `automation_analyze` に動的置換。
- [ ] 3. **ガードレール適用**: 解析結果をマスターデータと照合し、不該当データを排除（整合カテゴリ値の確定）。
- [ ] 4. **AI生成**: 属性に基づいた「最適化タイトル」および「商品説明本文」の自動生成。
    - **状態説明デフォルト**: 「致命的すぎるダメージではないと判断している為「目立った傷やよごれなし」でお出ししています。念の為、中古品にご理解ある方でお願い致します」を固定採用。
- [ ] 5. **ハッシュ選択**: `t3` (remarks) に基づき動的選択。
    - **デフォルトハッシュ**: メモなし等の場合 `00s_archive` を採用。
- [ ] 6. **マージ結合**: AI説明文 + テンプレート + ハッシュタグ + `purchase_id` を統合し「Master商品説明」として待機。

### Phase 3: 5点一括自動流し込み（Final Injection）
> 📢 **UI仕様**: 🅰️から🅱️、そして一括注入まで、一つのポップアップに情報を追記し続ける「永続ステータス表示」を採用（実装済）。
- [ ] **FINAL(1) : タイトル**: AIが生成・最適化した40文字以内のタイトル。

- [ ] **FINAL(1) : タイトル**: AIが生成・最適化した40文字以内のタイトル。
- [ ] **FINAL(2) : Master商品説明**: マージ済み・ID入りの完成版説明文。
- [ ] **FINAL(3) : 計算済みPrice**: 割引率を適用した数値。
- [ ] **FINAL(4) : 整合カテゴリ値**: マスター照合済みのカテゴリ設定。
- [ ] **FINAL(5) : メモ/Remarks**: タイトルから抽出されたメモ（DB保存および解析参照用）。
