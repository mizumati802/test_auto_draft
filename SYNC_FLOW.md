# 並列同期フロー & データマッピング仕様書 (SYNC_FLOW.md)

## 📌 オートメーション・アーキテクチャ
本プロジェクトは、フロントエンドでの「トリガー検知」から、2つの独立したリクエスト（タイトルRoot / 画像解析Root）を並列実行し、最終的に5つの重要データをメルカリへ一括注入する。

---

## 📅 実行フェーズ定義

### 🟩 Phase 1: トリガー & 初期解析 (Title Root 🅰️)
- **検知**: 下書き編集ページロード後「5秒」で自動発動。
- **Triple-T パース**: 
    - `t1`: 元値 (数値)
    - `t2`: 割引率 (数値のみ 3文字以内)
    - `t3`: 備考 (上記以外の全文字列)
- **バリデーション**: `t1` が数値でない、または画像がない場合は安全に停止。

### 🟨 Phase 2: データ・ブリッジ (Bridge 🤝)
- **役割**: タイトルRootから画像解析Rootへの文脈受け渡し。
- **Action**: `t3` を `remarks` (メモ) として画像解析リクエストのコンテキストに封入。
- **ID発行**: `purchase_id` の先行取得リクエスト。

### 🟧 Phase 3: 並列非同期処理 (Async Execution 🅱️)
- **AI解析 (Wrapper)**: 既存の `vintage_resolver` をラップして実行。
- **識別フラグ**: `analysis_type` を `automation_analyze` に動的置換。
- **生成物**: 
    - `title_textla`: AI生成のMasterタイトル。
    - `ai_description`: AI生成の商品説明本文。

### 🟦 Phase 4: 同期 & 最終マージ (Synchronization)
- **A & B 待ち合わせ**: 2つのRootの結果が揃ったことを検知。
- **マージロジック**:
    - **商品説明**: `ai_description` + テンプレート + ハッシュタグ + `purchase_id` を結合。
    - **価格算出**: `t1` と `t2` から **仕入れ価格** を算出。
    - **カテゴリ**: 出品理由(C1) + 解析結果(C2, C3) を統合。

### 🟪 Phase 5: 5点一括注入 (Final Injection 🚀)
メルカリの DOM および DB へ以下の5点を同時に反映する。

| 項目 | 物理カラム (DB) | メルカリ注入先 | 内容 |
| :--- | :--- | :--- | :--- |
| **FINAL(1)** | `title_textla` | 商品名 (40字) | AI最適化 Masterタイトル |
| **FINAL(2)** | `word_textla` | 商品説明 | マージ済み Master商品説明 |
| **FINAL(3)** | **`purchase_price`**| (DBのみ) | **仕入れ価格 (t1, t2由来)** |
| **FINAL(4)** | `category1-3` | カテゴリ選択 | C1:出品理由 / C2,C3:解析値 |
| **FINAL(5)** | **`memo`** | (DBのみ) | **備考 (t3の内容)** |

---

## 🛠 デフォルト・ガードレール仕様

### 1. 状態説明（Fixed Default）
商品説明の末尾には必ず以下の文言をセットする：
> 「致命的すぎるダメージではないと判断している為「目立った傷やよごれなし」でお出ししています。念の為、中古品にご理解ある方でお願い致します」

### 2. ハッシュタグ動的選択
- `t3` (remarks) 内のキーワードに基づきハッシュタグセットを選択。
- キーワードなし、または判定不能な場合は **`00s_archive`** をデフォルトとして採用。
